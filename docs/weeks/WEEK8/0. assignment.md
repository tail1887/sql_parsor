# WEEK8 — 미니 DBMS API 서버 (수요 코딩회)

## 목적

- 단 하루, AI를 도구로 삼아 팀별 프로젝트 구현에 몰입합니다.
- 결과물뿐 아니라 구현 과정을 중시합니다. AI가 생성한 코드도 핵심 로직은 반드시 이해하고 설명할 수 있어야 합니다.
- 낯선 기능을 Top-down으로 빠르게 분해·구현하며 학습 민첩성과 실무 감각을 높입니다.

## 요구사항

- 이번 주 과제는 **미니 DBMS API 서버** 구현입니다.
- 구현한 API를 통해 외부 클라이언트가 DBMS 기능을 사용할 수 있어야 합니다.
- **Thread Pool**을 구성하고, 요청마다 스레드를 할당해 SQL 요청을 병렬 처리해야 합니다.
- 이전 차수에서 구현한 **SQL 처리기**와 **B+ 트리 인덱스**를 내부 DB 엔진으로 재사용합니다.
- 구현 언어는 **C**입니다.
- 이번 과제는 "학습"보다 "구현"을 우선합니다. AI 도구를 적극 활용해 결과물을 완성하는 것을 최우선 목표로 합니다.
  - 단, 생성된 결과물의 핵심 로직은 반드시 이해하고 설명할 수 있어야 합니다.
  - 직접 코딩이 어렵다면 AI로 전체 코드를 생성해도 되지만, 원리 학습은 필수입니다.
- 다른 팀과 차별화할 수 있는 추가 구현 요소도 함께 고민해 보세요.

## 중점 포인트

- 멀티스레드 동시성 이슈 관리
- 내부 DB 엔진과 외부 API 서버의 연결 설계
- API 서버 아키텍처 설계

## 품질 기준

- 단위 테스트로 함수 단위 동작을 검증합니다.
- API 서버를 실행한 상태에서 기능 테스트를 통해 API 정상 동작을 확인합니다.
- 엣지 케이스를 최대한 고려합니다.
- 이력서/포트폴리오에 포함할 수 있을 수준의 완성도를 목표로 합니다.

## 결과물 발표

- 일정: **목요일 오전**
- 각 팀이 발표를 통해 프로젝트를 공유합니다.
- 팀 결과물을 간단히 데모합니다.
- 별도 발표자료 없이, GitHub 저장소의 `README.md`를 기준으로 설명합니다.
- AI로 빠르게 구현한 코드일수록 검증이 중요합니다.
  - 반드시 테스트 케이스를 작성하고 검증 과정을 수행합니다.
  - 검증 내용은 데모 발표에 포함합니다.
- 발표 시간: **4분**, Q&A: **1분**
